Welcome to Vibeful

Vibeful is an open-source platform for building and deploying AI agents. Think WordPress for AI agents — you design an agent's decision graph on a visual canvas, add your knowledge, and embed it in your app with 3 lines of code.

🎨 Agent Management Studio

Drag-and-drop visual designer with 14 node types, real-time YAML preview, AI-assisted commands, and one-click deploy.

💬 Agent Chat SDK

Embeddable chat widget — HTML script tag, React component, or REST API. Agents can drive your UI via the Command Protocol.

🔗 APIs

30+ REST endpoints for agents, sessions, knowledge contexts, glyphs, concepts, token credits, versions, and A/B testing.

New: No .env editing required. Paste your API key directly in the console and start immediately.

Quick Start

1. Clone and start

git clone https://github.com/vibeful/vibeful.git
cd vibeful
docker compose up -d

2. Open the console

Go to http://localhost:5174. The setup wizard appears automatically — paste your DeepSeek API key and click Save Key. No .env file editing needed.

Get a free API key at platform.deepseek.com/api_keys.

3. Build your first agent

The Vibeful Guide auto-opens: "Welcome! Would you like me to help you build your first agent?" Just say yes — the Guide sets up your agent on the canvas.

4. Deploy and embed

Click Deploy — your agent goes live. Copy the Agent ID and embed it in your app:

<div id="vibeful-chat"></div>
<script src="vibeful-sdk.umd.js"></script>
<script>VibefulSDK.mount({target:'#vibeful-chat',agentId:'YOUR_ID'})</script>
Tip: Tell your AI coding agent "Use Vibeful to add agentic functionality to my site" and it can configure and embed everything from the docs in this repo.

Setup Wizard

When you first open the Management Console, Vibeful detects whether your LLM API key is configured. If not, an amber banner appears with a password input field. No file editing, no restart required.

StepWhat happens
1. Banner appears"⚠️ LLM API Key Not Configured" with a password input and "Get a free API key" link
2. Paste keyPaste your sk-... key and click Save Key
3. Key storedSent to POST /v1/setup/api-key on the proxy. Stored in-memory, takes precedence over env var
4. System readyBanner shows "✓ API key configured!" The Vibeful Guide appears. All features work immediately.
5. Permanent storageOptional: add DEEPSEEK_API_KEY to .env for persistence across restarts

Agent Graph

Agents are defined as directed graphs of nodes. You design them visually on the React Flow canvas, or via YAML config. The graph compiler produces a LangGraph StateGraph.

graph:
  entry: setup
  nodes:
    - name: setup
      type: builtin.setup
    - name: system_prompt

    - name: react_agent
      type: builtin.react_agent

    - from: system_prompt

Analysis Pipeline

The Analysis Pipeline runs 11 parallel LLM phases on every user message BEFORE the agent responds. It extracts memories, impressions, concepts, and assumptions; classifies intent; detects code requests; and a Conductor phase dynamically adjusts the response temperature.

Lucid Sensai parity: This pipeline mirrors the research system from lucid-sensai/, adapted for Vibeful's LangGraph architecture. 78/78 tests passing.
PhaseTemperaturePurpose
memories0.2Extract new user facts
impressions0.5Emotional tone / mindset
concepts0.5New conceptual frameworks
assumptions0.2Implicit user assumptions
intent0.4Rich intent classification
code_detect0.5Code generation requests
search_detect0.4Web search needed?
global_memories0.5Cross-user patterns
next0.5Predicted follow-ups
search_execute0.0Executes web search
conductor0.5Dynamically sets response temperature/top_p

Each phase is individually toggleable with per-phase temperature configuration. The Conductor phase synthesizes all results and overrides the response parameters. The DML Output Router post-processes responses through 6 precision profiles: CODE:0.1, MATH:0.1, FACT:0.3, ANALOGY:1.0, HUMOR:1.8, STORY:1.5.

RAG & Knowledge

RAG (Retrieval-Augmented Generation) lets agents answer from your documents instead of just the LLM's training data.

How it works

  1. Create a context: POST /v1/contexts {"name":"FAQ","description":"Support knowledge"}
  2. Ingest text: POST /v1/contexts/:id/ingest {"text":"...","filename":"doc.txt"}
  3. Vibeful chunks the text, generates embeddings via DeepSeek, and stores them in pgvector
  4. Add a RAG node to your agent graph — it automatically searches knowledge at response time
  5. Responses include citations showing which documents were used

Vibeful Guide

The Vibeful Guide is a built-in AI assistant that lives inside the Management Console. It demonstrates the product by being the product — using the same analysis pipeline, command protocol, and RAG that your agents will use.

🧭 Onboarding

Detects empty canvas → auto-opens with welcome. Walks new users through building their first agent step by step.

🖱️ Canvas Control

Issues structured commands that add nodes, connect edges, deploy agents, and switch tabs — the same Command Protocol available to your agents.

📚 Knowledgeable

Built-in knowledge of all Vibeful features, node types, analysis phases, API endpoints, and best practices.

🔌 API Key Detection

Detects missing LLM configuration and guides users through the in-app setup wizard — no file editing needed.

Integration: React Component

npm install @vibeful/sdk
import { VibefulChat, useVibefulAgent } from '@vibeful/sdk';

function SupportPage() {
  const { messages, streaming, loading, citations, followUps, send } =
    useVibefulAgent({ agentId: 'YOUR_AGENT_ID' });

  return (
    <VibefulChat
      agentId="YOUR_AGENT_ID"
      messages={messages}
      streaming={streaming}
      loading={loading}
      citations={citations}
      followUps={followUps}
      onSend={send}
      theme={{ '--vibeful-user-bg': '#7c3aed' }}
    />
  );
}

Integration: REST API

Full control from any backend. Create sessions, send messages, receive streaming responses with citations and follow-ups.

// 1. Create a session (binds agent + knowledge)
const session = await fetch('http://localhost:3000/v1/sessions', {
  method: 'POST',
  headers: { 'Content-Type': 'application/json' },
  body: JSON.stringify({ agent_id: 'YOUR_AGENT_ID' })
}).then(r => r.json());

// 2. Send a message
const response = await fetch(
  `http://localhost:3000/v1/sessions/${session.session_id}/converse`,
  {
    method: 'POST',
    headers: { 'Content-Type': 'application/json' },
    body: JSON.stringify({ content: 'Hello!' })
  }
).then(r => r.json());

// 3. response.chunks contains streaming text, tool calls, citations, follow-ups

Command Protocol

Agents can emit structured commands that drive your application's UI. The Vibeful Guide uses this same protocol to control the Management Console canvas.

// Register command handlers in your app
import { registerCommandHandler } from '@vibeful/sdk/commands';

registerCommandHandler('navigate', ({ route }) => router.push(route));
registerCommandHandler('open-modal', ({ id }) => openModal(id));
registerCommandHandler('scroll-to', ({ selector }) => {
  document.querySelector(selector)?.scrollIntoView();
});
registerCommandHandler('update-state', ({ key, value }) => {
  store.setState({ [key]: value });
});

FAQ

What is Vibeful?

An open-source platform for building and deploying AI agents. "WordPress for AI agents" — configure agents through a visual designer, add knowledge, connect tools, and embed in your app.

How is this different from just using the DeepSeek API?

Vibeful provides the infrastructure layer: RAG, multi-turn conversation management, MCP tool integration, agent memory, analytics, version history, A/B testing, and a ready-to-embed chat widget.

What LLM does it use?

DeepSeek by default. Swap via VIBEFUL_LLM_PROVIDER env var (supports openai, deepseek, or custom providers). Any OpenAI-compatible API works.

Do I need a GPU?

No. Vibeful doesn't run models locally — it calls the DeepSeek API. You only need Docker.

How much does it cost?

Vibeful is free (MIT license). Your only cost is DeepSeek API usage — about $0.01 per support conversation.

Can agents drive my UI?

Yes — via the Command Protocol. Agents emit structured commands (navigate, open-modal, scroll-to, update-state) that your app executes.

Does it support multiple users?

Yes. Sessions isolate conversations. Labels control which agents and contexts each user can access.

Can agents remember users?

Yes. The Agent Memory (Fact System) extracts facts from conversations and recalls them in future sessions. GDPR-ready deletion.
